# Favorites Page Data

The favorites page is driven by hand-edited config plus generated metadata:

- `favorites.json` is the source of truth for which films and books appear, plus any manual ordering or overrides.
- `data/metadata.json` is generated by `fetch_covers.py`. Do not hand-edit it.
- `src/pages/favorites.astro` imports both files, resolves each favorite, and renders separate desktop and mobile grids.

Run all commands from the repo root with the virtualenv active:

```bash
source .venv/bin/activate
python scripts/favorites/fetch_covers.py
npm run build
```

## Adding A Favorite

Add the URL to the matching top-level list in `favorites.json`:

```json
{
  "films": [
    "https://letterboxd.com/film/example-film/"
  ],
  "books": [
    "https://www.goodreads.com/book/show/60811826"
  ]
}
```

Then run `python scripts/favorites/fetch_covers.py`. It scrapes Letterboxd and Goodreads, then rewrites `scripts/favorites/data/metadata.json` with title, subtitle, and cover data for each URL.

If `orders` are present for the relevant viewport, also add the URL there. An item that is in the top-level list but missing from an explicit order list will not appear in that viewport.

## Changing Order

Use the `orders` block in `favorites.json`.

```json
{
  "orders": {
    "desktop": {
      "films": [
        "https://letterboxd.com/film/spirited-away/"
      ],
      "books": [
        "https://www.goodreads.com/book/show/13641208-tenth-of-december"
      ]
    },
    "mobile": {
      "films": [
        "https://letterboxd.com/film/spirited-away/"
      ],
      "books": [
        "https://www.goodreads.com/book/show/60811826"
      ]
    }
  }
}
```

Desktop and mobile order lists are independent. Keep each list complete for its viewport unless the goal is to hide an item on that viewport.

## Different Desktop And Mobile Lists

The top-level `films` and `books` arrays define the available pool. The viewport order lists decide what each viewport renders.

- To show an item on both desktop and mobile: include it in the top-level list, `orders.desktop`, and `orders.mobile`.
- To show an item only on mobile: include it in the top-level list and `orders.mobile`, but omit it from `orders.desktop`.
- To show an item only on desktop: include it in the top-level list and `orders.desktop`, but omit it from `orders.mobile`.

This works because `favorites.astro` builds separate `desktopFilms`, `mobileFilms`, `desktopBooks`, and `mobileBooks` arrays.

## Overriding Covers Or Posters

Use an object instead of a bare URL when you need a specific cover, poster, title, or subtitle:

```json
{
  "url": "https://www.goodreads.com/book/show/60811826",
  "title": "I Who Have Never Known Men",
  "subtitle": "Jacqueline Harpman",
  "cover": "/images/favorites/books/i-who-have-never-known-men.jpg"
}
```

For films, use the same shape:

```json
{
  "url": "https://letterboxd.com/film/arrival-2016/",
  "cover": "/images/favorites/films/arrival.jpg"
}
```

Local assets should live under:

- `public/images/favorites/books/`
- `public/images/favorites/films/`

Reference them with public paths like `/images/favorites/books/example.jpg`.

`fetch_covers.py` always scrapes title/subtitle/cover metadata, but if a favorite object has a `cover`, that cover is copied into generated `metadata.json`. `favorites.astro` also lets values in `favorites.json` override generated metadata at render time, so `title` and `subtitle` fields are useful for canonical names or display tweaks.

## Optimizing Remote Images

`optimize_assets.py` is optional. It reads generated metadata, downloads remote covers, writes optimized local images into `public/images/favorites/`, and rewrites `metadata.json` to point at local files.

Run it only when you intentionally want to localize or refresh image assets:

```bash
source .venv/bin/activate
python scripts/favorites/optimize_assets.py
```

After either script, run `npm run build` and verify `/favorites/` contains the changed title, URL, and cover path.
